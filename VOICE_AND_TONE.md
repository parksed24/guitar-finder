# Guitar Finder Voice & Tone Guide

## Brand personality

Guitar Finder should feel like a knowledgeable guitar-shop friend who is helping someone find the right instrument quickly.

The product should feel:

- enthusiastic
- conversational
- informed
- trustworthy
- efficient
- modern
- lightly playful

The product should not feel:

- sterile
- corporate
- overly technical
- salesy
- gimmicky
- overly casual
- verbose
- patronizing

Buying a guitar is exciting. The interface should reflect that while staying easy to scan.

## Core tone

Write like someone who genuinely knows guitars and enjoys helping people find the right one.

Good:

> Let’s find the right one.

> A few strong matches surfaced.

> This one is worth a closer look.

> Nothing exact yet. Want us to keep an eye out?

Avoid:

> No purchasable individual guitar listings passed the exact-match filters.

> Search request completed successfully.

> External marketplace discovery results.

> Canonical product validation failed.

The system can be technically sophisticated internally. The user-facing copy should stay simple.

## Writing principles

### 1. Be clear before clever

Use familiar language.

Prefer:

> No exact matches yet.

Avoid:

> No qualified listings met the canonical match threshold.

### 2. Keep copy short

Most UI text should be one sentence or less.

Prefer:

> Price available on source

Avoid:

> Pricing information was not available in the search result. Open the original source page to verify the current listing price.

### 3. Make guitar shopping feel fun

Use light enthusiasm when it helps.

Prefer:

> Nice choice.

> We found a few good ones.

> This one looks promising.

> Worth a closer look.

Avoid excessive hype:

> Incredible deal!

> Insane guitar!

> Absolute must-buy!

### 4. Be honest about uncertainty

Never imply certainty when the source data is incomplete.

Prefer:

> Likely exact match

> Possible match

> Availability unverified

> Price available on source

> Open the listing to confirm details

Avoid:

> Verified listing

unless the app has actually verified the listing through structured source data.

### 5. Do not expose internal implementation language

Avoid:

- canonical query
- search orchestration
- metadata enrichment
- source-backed extraction
- confidence threshold
- normalized listing
- purchasability classifier
- web discovery lane
- source registry
- API result
- external module

Translate internal concepts into plain language.

### 6. Avoid generic ecommerce language

The app should not feel like a generic marketplace template.

Avoid:

> Discover products

> Browse inventory

> Shop now

> Product details

Prefer:

> Find your guitar

> View listing

> See what’s out there

> Take a closer look

## Voice by surface

### Home page

Goal: exciting, simple, low-pressure.

Recommended:

**Headline**

> Find the guitar you’ve been looking for.

**Supporting copy**

> Search shops, marketplaces, and classifieds in one place.

**Search placeholder**

> Search model, finish, artist signature, or SKU

**Advisor CTA**

> Not sure what to buy?

> Answer a few questions and we’ll narrow it down.

### Search results

Goal: focused and scannable.

Use:

> Results

> 12 likely matches found so far

> Refresh search

> Reset feed

> Save search

> Load more results

Avoid:

> Unified listing feed

> Web results

> Qualified candidates

> Search additional platforms

For fallback links, use:

> Still looking?

> Check a few more places directly.

### Empty states

Use:

> No exact matches yet.

> We didn’t find the exact guitar this time.

> Want us to keep an eye out?

> Save this search and we’ll let you know when one turns up.

Avoid:

> No purchasable individual guitar listings passed the exact-match filters.

### Listing cards

Use:

> Likely exact match

> Possible match

> Availability unverified

> Price available on source

> Local pickup

> Ships to you

> View listing

Avoid:

> Strong web match

> Open web

> listing detail

> Web result

> source-backed metadata

### Listing detail page

Use:

> Total estimated cost

> Cost breakdown

> About this listing

> Listing details

> Purchase details

> Open on Reverb

> Open on Ish Guitars

> Open original listing

### Advisor

Goal: friendly, quick, confident.

Use:

> Let’s narrow it down.

> What do you want to play?

> Who inspires your sound?

> What tuning do you use?

> Are you open to used guitars?

> Your strongest matches

> See listings for this guitar

> Edit answers

Avoid:

> Recommendation engine

> Optimize results

> Buying lane

> Authentication required

### Saved searches and alerts

Use:

> Save this search

> Want us to keep an eye out?

> Get notified when a new match shows up.

> Track this guitar

> Price-drop alerts

> New-listing alerts

> Saved

Avoid:

> Configure notification preferences

> Alert criteria

> Saved query

| Web results | Results |

| External search | Check another site |
| Search additional platforms | Still looking? |
| Verified model matches | Likely matches |
| Product details | Listing details |
| Itemized estimate | Cost breakdown |
| Landed total | Total estimated cost |
| Reset results | Reset feed |
| Recommendation result | Match |
| Saved queries | Saved searches |
| Authentication | Sign in |

## Button copy

Prefer direct, human CTAs:

- Search
- View listing
- See listings
- Save search
- Track this guitar
- Refresh search
- Load more results
- Edit answers
- Back to results
- Add to compare
- Open on Reverb
- Open original listing
- Clear filters
- Show results

Avoid vague CTAs:

- Continue
- Submit
- Proceed
- Learn more
- Execute search
- View web result

## Error messages

Errors should be calm and useful.

Use:

> Something went wrong while searching. Try again.

> We couldn’t load more results. Give it another shot.

> That search didn’t return anything useful yet.

> We couldn’t confirm the details on this listing. Open the source to double-check.

Avoid:

> API error

> Search pipeline failure

> Invalid canonical query

> Source enrichment failed

## Copy length rules

- Headline: 3–8 words
- Supporting text: 1 short sentence
- Buttons: 1–4 words
- Tags: 1–3 words
- Empty states: 1–2 short sentences
- Listing cards: no paragraphs
- Detail-page helper copy: no more than 2 short sentences

## Personality guardrails

A little personality is good.

Good:

> Nice choice.

> We found a few promising ones.

> Nothing exact yet.

> This one is worth a closer look.

Do not overdo it.

Avoid:

> Sick axe!

> Shred machine!

> Tone monster!

> Dude, this guitar rips!

## Final rule

Whenever user-facing copy feels technical, corporate, or overly formal, rewrite it in simpler language.

The product should feel like a smart guitar-shopping companion, not a database interface.
